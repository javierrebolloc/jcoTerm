## Qué se implementó

### 1. tasks-improvements.md — 10/10 tareas completadas

- **UX-01**: Streaming de respuestas IA (Anthropic + Gemini via IPC push events)
- **UX-02**: Loading states en conexión SSH (spinner en SessionItem)
- **UX-03**: Mensajes de error más informativos (SFTP con ruta, Gemini con reset time, SSH Agent)
- **UX-04**: Modelo Anthropic configurable en Ajustes (Sonnet/Haiku/Opus)
- **UX-05**: Auto-recovery de sesión tras crash (window-state.json)
- **FEAT-01**: SSH Agent integration (Windows OpenSSH pipe)
- **FEAT-02**: max_tokens dinámico (1024-4096 según contexto)
- **FEAT-03**: Log de redacción detallado (matchedTypes)
- **DEVOPS-01**: npm audit script
- **DEVOPS-02**: @anthropic-ai/sdk 0.105.0, electron-store ^8.2.0

### 2. Rediseño de Ajustes con sidebar

- Modal 700px con sidebar izquierdo y 5 secciones: General, Terminal, IA, Conexiones, Acerca de
- Nuevos settings: fontFamily, cursorStyle, cursorBlink
- Settings se aplican en vivo al terminal via `term.options.*`
- Selector de idioma funcional
- About: autor, email, licencia MIT, versión

### 3. Gestor SFTP (FileZilla-like)

**Backend:**
- 8 nuevos métodos SFTP en SshSession: stat, mkdir, rmdir, unlink, rename, chmod, download (streaming), upload (streaming)
- `formatPermissions()` — mode Unix a string
- local.handlers.ts: listDir, homePath, drives (Windows A:-Z:)

**Frontend (16 componentes):**
- SftpManager — contenedor con multi-tab remoto, error toast
- SftpConnectionBar — selector sesión + conectar/desconectar
- SftpFilePane — panel reutilizable (local/remoto) con toolbar
- SftpFileTable — tabla con multi-select (Ctrl/Shift), sorting, búsqueda por teclado, drag & drop
- SftpBreadcrumb — navegación por ruta con auto-scroll
- SftpContextMenu — menú contextual
- SftpTransferQueue — cola con progreso y errores
- SftpChmodDialog — editor 3x3 + octal

**Features:**
- Dual-pane: local (izq) + remoto (der)
- Múltiples conexiones remotas con pestañas
- Upload/download con progreso en tiempo real
- Drag & drop entre paneles (multi-fichero)
- Multi-selección (Ctrl+click, Shift+click)
- Búsqueda por teclado (escribe letras para buscar)
- mkdir, delete, rename, chmod
- Diálogo de credenciales si la sesión no tiene password
- Errores de transferencia mostrados en cola y toast

### 4. Sistema i18n

- Motor propio: `t(key, params?)`, `setLocale()`, `getLocale()`
- JSON plano con claves dot-separated (~250 keys)
- `en.json` (inglés, default) + `es.json` (español)
- React: LanguageProvider + useTranslation hook
- Main process: import directo de `t()`
- ~20 componentes renderer actualizados
- ~12 handlers main actualizados
- Aviso de reinicio al cambiar idioma

### 5. Multi-ejecución en split

- Opción "Multi-ejecución" en menú Split
- Input replicado a todos los terminales del split
- Cada terminal tiene toggle ">>" para excluirse
- Se desactiva automáticamente al salir de split

### 6. Diálogos de confirmación custom

- ConfirmProvider + useConfirm() reemplazan `window.confirm()`
- Tema oscuro, botón rojo para acciones destructivas
- 5 puntos de uso: eliminar sesión/carpeta/credencial/fichero SFTP, aceptar host key

### 7. Bugfixes

- Pantalla negra: `handleConnect` referenciado antes de inicialización (temporal dead zone)
- xterm.js dimensions: try-catch con retry en `term.open()`
- Upload 0%: handler devolvía transferId DESPUÉS del stream (ahora en background)
- Chmod input: filtro de caracteres no-octales, sin loop de sync
- ConnectDialog: CSS classes desalineadas (.dialog vs .modal)
- Modales no se cierran al click fuera (intencional, por petición del usuario)
- autoFocus en campo usuario (no en password)
- Quitado emoji llave de Credenciales, quitado "+" de Nueva sesión
- Error Autofill.enable suprimido (quitado auto-open DevTools)
- Paste multi-exec: Shift+right-click pega en todos los terminales del split
- Terminal cortada en split: padding reducido, statusBar compacta, min-width/min-height en celdas grid
