## APIs and Performance — 17 of 22 Tasks Resolved

- **GeminiClient**: retry on 500/503, future resetAt validation, log without "unknown"
- **SshSession**: shell timeout 10s, persistent `client.on('error')` + `stream.on('error')`, SFTP close race fix, write backpressure, resize validation
- **xterm.js**: ResizeObserver debounce 100ms, configurable scrollback from settings, clipboard error logging, getVisibleContent limited to 2000 lines

## Bugfixes Found During Manual Testing

1. **Split view black screen**: TerminalPanes were unmounting/remounting when switching from normal to split mode (different conditional JSX branches). Refactored so panes render in a single container with layout controlled by CSS.

2. **Split menu disappears**: 4px gap between button and menu caused `onMouseLeave` to fire when moving the mouse. Fix: `top: 100%` + `padding-top: 4px` (transparent padding inside the hover wrapper).

3. **`config is not defined` in ssh:connect catch**: Variable `config` declared with `let` inside the `try` was not accessible in the `catch` after bundling. Fix: `connectHost`/`connectPort` variables outside the try-catch.

4. **Quota bar "no data"**: The UI showed "Daily quota: no data" when Gemini didn't send quota headers. Fix: only show bar when `remaining !== null`.

5. **Log "remaining quota: unknown"**: Changed to omit the quota part in the log when there's no data.
