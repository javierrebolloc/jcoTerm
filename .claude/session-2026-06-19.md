## Qué se implementó hoy

### 1. Mejoras al GeminiClient (`src/main/ai/GeminiClient.ts`)

**Logging detallado de 429s:**
- Antes: solo se logeaba un resumen al final de todos los reintentos
- Ahora: cada intento 429 logea `log.warn` con el mensaje de error + `log.debug` con el JSON completo de la respuesta

**`detectLimitType` corregido:**
- Bug anterior: el mensaje de error del usuario contenía `generate_content_free_tier_requests` Y `generate_content_free_tier_input_token_count`. El código comprobaba "token" antes que "free_tier_requests" → clasificaba como `tpm` (incorrecto)
- Fix: `free_tier_requests` y `requests_per_day` se comprueban ANTES que `token`

**`parseRetryAfterMs` (nueva función):**
- Lee el delay sugerido por la API en el header `Retry-After` o en el patrón `"Please retry in X.Xs"` del mensaje de error
- Cappado a `MAX_API_RETRY_DELAY_MS = 60_000` ms
- Si existe, overrida el delay de backoff exponencial para el siguiente reintento

**`isZeroQuotaKey` (nueva función) + fast-fail:**
- Detecta cuando todos los `limit: N` del error tienen N=0 (regex: `/limit: 0/` presente y `/limit: [1-9]/` ausente)
- Si es verdad, lanza `GeminiQuotaError` inmediatamente sin reintentar (antes esperaba 30s + 60s + 60s = ~150s inútiles)
- Log especial: "La clave API tiene cuota 0 en todos los limites — verifica la configuracion del proyecto en Google AI Studio"

**Modelo por defecto cambiado:**
- `gemini-2.0-flash-lite` → `gemini-2.5-flash-lite`
- Motivo: el proyecto de Google del usuario tenía cuota 0 para 2.0-flash-lite pero 0/10 RPM + 0/250K TPM + 0/20 RPD para 2.5-flash-lite
- Cambiado en: `GeminiClient.ts` constructor default, `SettingsStore.ts` defaults, `SettingsModal.tsx` lista de modelos (2.5-flash-lite añadido como primera opción y default)

**Fix encoding:**
- `…` (U+2026) en mensajes de log → `...` para evitar que aparezca como `ÔÇª` en logs Windows

**6 tests nuevos** en `gemini-client.test.ts`:

- Mensaje con `free_tier_requests` + `input_token_count` → `daily` (no `tpm`)

### 4. Copy/paste en terminal (`src/renderer/components/Terminal/useTerminal.ts`)

Añadido en el `useEffect` principal (mount por `sessionId`):

```typescript
// COPY: mouseup → getSelection → clipboard.writeText
const handleMouseUp = (): void => {
  const selection = term.getSelection()
  if (selection) void navigator.clipboard.writeText(selection)
}
container.addEventListener('mouseup', handleMouseUp)

// PASTE: contextmenu → clipboard.readText → sendInput
const handleContextMenu = (e: MouseEvent): void => {
  e.preventDefault()
  void navigator.clipboard.readText().then((text) => {
    if (text) window.electronAPI.ssh.sendInput(sessionId, text)
  }).catch(() => {})
}
container.addEventListener('contextmenu', handleContextMenu)
```

Limpieza en el return del `useEffect`:
```typescript
container.removeEventListener('mouseup', handleMouseUp)
container.removeEventListener('contextmenu', handleContextMenu)
```

Nota: `copyOnSelect: true` de xterm.js NO se usa — usa `document.execCommand('copy')` que falla silenciosamente en Electron. La solución correcta es `navigator.clipboard.writeText()` manual.
