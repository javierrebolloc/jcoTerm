## What Was Implemented Today

### 1. GeminiClient Improvements (`src/main/ai/GeminiClient.ts`)

**Detailed 429 logging:**
- Before: only a summary was logged at the end of all retries
- Now: each 429 attempt logs `log.warn` with the error message + `log.debug` with the full JSON response

**`detectLimitType` fixed:**
- Previous bug: the user error message contained `generate_content_free_tier_requests` AND `generate_content_free_tier_input_token_count`. The code checked "token" before "free_tier_requests" → classified as `tpm` (incorrect)
- Fix: `free_tier_requests` and `requests_per_day` are checked BEFORE `token`

**`parseRetryAfterMs` (new function):**
- Reads the suggested delay from the API in the `Retry-After` header or in the `"Please retry in X.Xs"` pattern from the error message
- Capped at `MAX_API_RETRY_DELAY_MS = 60_000` ms
- If present, overrides the exponential backoff delay for the next retry

**`isZeroQuotaKey` (new function) + fast-fail:**
- Detects when all `limit: N` in the error have N=0 (regex: `/limit: 0/` present and `/limit: [1-9]/` absent)
- If true, throws `GeminiQuotaError` immediately without retrying (before it waited 30s + 60s + 60s = ~150s uselessly)
- Special log: "The API key has quota 0 on all limits — check the project configuration in Google AI Studio"

**Default model changed:**
- `gemini-2.0-flash-lite` → `gemini-2.5-flash-lite`
- Reason: the user's Google project had quota 0 for 2.0-flash-lite but 0/10 RPM + 0/250K TPM + 0/20 RPD for 2.5-flash-lite
- Changed in: `GeminiClient.ts` constructor default, `SettingsStore.ts` defaults, `SettingsModal.tsx` model list (2.5-flash-lite added as first option and default)

**Encoding fix:**
- `...` (U+2026) in log messages → `...` to avoid appearing as `ÔÇª` in Windows logs

**6 new tests** in `gemini-client.test.ts`:
- `free_tier_requests` → classified as `daily`
- Message with `free_tier_requests` + `input_token_count` → `daily` (not `tpm`)
- Fast-fail with `limit: 0` → only 1 fetch call (not 4)
- No fast-fail when some limit > 0 → 4 normal calls
- Retry respecting "Please retry in 0.001s"
- Retry respecting header `Retry-After: 0.001`

### 4. Copy/paste in terminal (`src/renderer/components/Terminal/useTerminal.ts`)

Added in the main `useEffect` (mount by `sessionId`):

```typescript
// COPY: mouseup → getSelection → clipboard.writeText
const handleMouseUp = (): void => {
  const selection = term.getSelection()
  if (selection) void navigator.clipboard.writeText(selection)
}
container.addEventListener('mouseup', handleMouseUp)

// PASTE: contextmenu → clipboard.readText → sendInput
const handleContextMenu = (e: MouseEvent): void => {
  e.preventDefault()
  void navigator.clipboard.readText().then((text) => {
    if (text) window.electronAPI.ssh.sendInput(sessionId, text)
  }).catch(() => {})
}
container.addEventListener('contextmenu', handleContextMenu)
```

Cleanup in the `useEffect` return:
```typescript
container.removeEventListener('mouseup', handleMouseUp)
container.removeEventListener('contextmenu', handleContextMenu)
```

Note: `copyOnSelect: true` from xterm.js is NOT used — it uses `document.execCommand('copy')` which fails silently in Electron. The correct solution is manual `navigator.clipboard.writeText()`.
